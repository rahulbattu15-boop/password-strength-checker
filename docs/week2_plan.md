# Week 2 Plan

## Core Feature 1: Password Scoring

I need to make a system that checks how good a password is. This system will give the password a score out of 100. The score will depend on how long the password's if it has lowercase letters, uppercase letters, numbers, special characters and if it is a common password. It will also check for repeated characters and simple sequences like abc or 123. The password scoring system will look at all these things to give the password a score.

## Core Feature 2: Password Suggestions

If a password is weak or medium I will give the user some tips to make it better. These tips will include making the password adding numbers to the password adding special characters to the password and not using common passwords or simple patterns like your name or birthday. The password suggestions will help users create a password.

## Core Feature 3: Browser Demo

I will make a web page where users can try out the password checker. They can type in a password. See right away how good it is. The demo will not. Send the password anywhere. This way users can test their passwords without worrying about security. The browser demo is, for testing the password scoring system.

## Testing Plan

We will test the password checker with kinds of passwords including:

- No password at all

- A very short password

- A common password that many people use

- A password that only has letters

- A password that has letters and numbers

- A strong password that has letters, numbers and special characters. The password checker needs to work with all these types of passwords.
